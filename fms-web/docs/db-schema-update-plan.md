## 3. 구현 순서

1. **1단계: DB 스키마 수정**
   - ALTER TABLE 실행 (기존 테이블 컬럼 추가)
   - CREATE TABLE 실행 (신규 테이블 생성)

2. **2단계: API 수정**
   - 각 API route.ts 파일의 INSERT/UPDATE 쿼리 수정
   - 새로 추가된 컬럼 반영

3. **3단계: 등록 페이지 handleSave 수정**
   - 화면 폼 데이터를 API로 전달하는 부분 수정
   - 새 필드들 포함

4. **4단계: 저장 테스트**
   - 각 등록 페이지에서 테스트 데이터 입력
   - DB 저장 확인
   - 조회 시 데이터 복원 확인
