# FMS Database Schema Overview

## 스키마 구조

FMS(Forwarder Management System) 데이터베이스는 다음 도메인으로 구성됩니다:

```
┌─────────────────────────────────────────────────────────────────┐
│                        FMS Database                              │
├─────────────────────────────────────────────────────────────────┤
│  01. Master (기준정보)                                           │
│      - 회사, 화주, 실행사, 파트너, 항구/공항, 국가, 통화 등       │
├─────────────────────────────────────────────────────────────────┤
│  02. Schedule (스케줄)                                           │
│      - 해상 스케줄, 항공 스케줄, SPACE 관리                       │
├─────────────────────────────────────────────────────────────────┤
│  03. Order & Booking (오더/부킹)                                 │
│      - 수출/수입 오더, 해상/항공 부킹                             │
├─────────────────────────────────────────────────────────────────┤
│  04. B/L & AWB (선하증권/항공운송장)                             │
│      - HBL, MBL, HAWB, MAWB, 컨테이너, 화물                      │
├─────────────────────────────────────────────────────────────────┤
│  05. Shipment (선적/도착)                                        │
│      - S/R, S/N, A/N, Pre-Alert                                  │
├─────────────────────────────────────────────────────────────────┤
│  06. Inland Transport (내륙운송)                                 │
│      - 운송지시, 운송스케줄, POD/GR                               │
├─────────────────────────────────────────────────────────────────┤
│  07. Customs (통관)                                              │
│      - 수출/수입 통관, AMS, 적하목록                              │
├─────────────────────────────────────────────────────────────────┤
│  08. Billing (정산)                                              │
│      - 계약, 요율(Tariff), 매출/매입, Invoice                     │
└─────────────────────────────────────────────────────────────────┘
```

## 네이밍 규칙

### 테이블명
- 접두어: 도메인별 3자리 (MST_, SCH_, ORD_, BIL_, SHP_, TRN_, CUS_, BIL_)
- 형식: `{PREFIX}_{ENTITY_NAME}`
- 예: `MST_CUSTOMER`, `ORD_BOOKING`, `BIL_INVOICE`

### 컬럼명
- PK: `{TABLE}_ID` 또는 `{TABLE}_NO`
- FK: 참조 테이블의 PK명 그대로 사용
- 일자: `*_DT` (DATE), `*_DTM` (DATETIME)
- 코드: `*_CD`
- 명칭: `*_NM`
- 수량: `*_QTY`
- 금액: `*_AMT`
- 중량: `*_WGT`
- 여부: `*_YN`

### 공통 컬럼
모든 테이블에 포함되는 공통 감사(Audit) 컬럼:
```sql
CREATED_BY      VARCHAR(50)     -- 생성자
CREATED_DTM     DATETIME        -- 생성일시
UPDATED_BY      VARCHAR(50)     -- 수정자
UPDATED_DTM     DATETIME        -- 수정일시
DEL_YN          CHAR(1)         -- 삭제여부 (Y/N)
```

## 데이터베이스 설정

- DBMS: PostgreSQL 15+ (또는 MySQL 8+)
- Character Set: UTF-8
- Collation: utf8mb4_unicode_ci (MySQL) / en_US.UTF-8 (PostgreSQL)
